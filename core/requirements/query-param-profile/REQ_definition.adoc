[[req_query-param-profile_definition]]
[requirement]
====
[%metadata]
identifier:: /req/query-param-profile/definition

[.component,class=part]
--
A server implementation SHALL support the Profile (`profile`) parameter for the GET operation.
--

[.component,class=part]
--
The following OpenAPI 3.0 schema fragment SHALL define the characteristics of the `profile` parameter: 

[source,YAML]
----
name: profile
in: query
required: false
schema:
  type: array
  items:
    type: string
explode: false
style: form
----
--

[.component,class=part]
--
Each item SHALL be one of the following:

* A HTTP(S) URI of a profile, e.g., in the OGC Profile Register (\https://www.opengis.net/def/profile);
* A Safe CURIE of a profile in the OGC Profile Register (e.g., `[ogc-profile:my-profile]`);
* A profile identifier (the `profileId` value in the URI template `\https://www.opengis.net/def/profile/OGC/0/{profileId}`)
--
