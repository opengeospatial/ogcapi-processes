[[ats_kvp-execute_output]]

[abstract_test]
====
[%metadata]
identifier:: /conf/kvp-execute/output
target:: /req/kvp-execute/output
test-purpose:: Verify that requested outputs on a KVP-encoded execute request are handled correctly.
test-method::
+
--
1. Get a list of processes as per test <<ats_core_process-list-op,/conf/core/process-list-op>>.

2. Retrieve the description of each process according to test <<ats_core_process-description-op,/conf/core/process-description-op>>.

3. Select a process to execute.

4. Inspect the output definitions of the process taking note of the defined characetristics of each output (e.g. whether more that one output format is supported, etc.).

5. Compose a KVP-encoded execute request encoding each requested output according to <<req_kvp-execute_output,/req/kvp-execute/output>>.

6. Execute the request using the HTTP `GET` method.

7. Verify that the response of the process is the expected response (see tests: <<req_core_process-execute-success-async,/req/core/process-execute-success-async>>, <<req_core_process-execute-success-async-outputs,/req/core/process-execute-success-async-outputs>>, <<req_core_process-execute-success-async-outputs-empty,/req/core/process-execute-success-async-outputs-empty>>, <<req_core_process-execute-success-async-outputs-omitted,/req/core/process-execute-success-async-outputs-omitted>>, <<req_core_process-execute-success-sync-empty-outputs,/req/core/process-execute-success-sync-empty-outputs>>, <<req_core_process-execute-sync-many-json,/req/core/process-execute-sync-many-json>>, <<req_core_process-execute-sync-one,/req/core/process-execute-sync-one>>, <<req_core_process-execute-sync-one-default-content,/req/core/process-execute-sync-one-default-content>>).
--
====
