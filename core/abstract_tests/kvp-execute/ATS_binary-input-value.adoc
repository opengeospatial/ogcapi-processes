[[ats_kvp-execute_binary-input-value]]

[abstract_test]
====
[%metadata]
identifier:: /conf/kvp-execute/binary-input-value
target:: /req/kvp-execute/binary-input-value
test-purpose:: Verify that binary-valued process inputs for KVP-encoded execute request are correctly handled.
test-method::
+
--
1. Get a list of processes as per test <<ats_core_process-list-op,/conf/core/process-list-op>>.

2. Retrieve the description of each process according to test <<ats_core_process-description,/conf/core/process-description>>.

3. Inspect the input definitions of each process and identify a process that has at least one binary-typed input that DOES NOT need to be <<qualified-value-schema,qualified>> with a <<format-schema,format>> specification.

4. Compose a KVP-encoded execute request encoding the binary-typed input(s) according to <<req_kvp-execute_binary-input-value,/req/kvp-execute/binary-input-value>>.

5. Execute the request using the HTTP `GET` method.

6. Verify that the response of the process is the expected response (see tests: <<req_core_process-execute-success-async,/req/core/process-execute-success-async>>, <<req_core_process-execute-success-async-outputs,/req/core/process-execute-success-async-outputs>>, <<req_core_process-execute-success-async-outputs-empty,/req/core/process-execute-success-async-outputs-empty>>, <<req_core_process-execute-success-async-outputs-omitted,/req/core/process-execute-success-async-outputs-omitted>>, <<req_core_process-execute-success-sync-empty-outputs,/req/core/process-execute-success-sync-empty-outputs>>, <<req_core_process-execute-sync-many-json,/req/core/process-execute-sync-many-json>>, <<req_core_process-execute-sync-one,/req/core/process-execute-sync-one>>, <<req_core_process-execute-sync-one-default-content,/req/core/process-execute-sync-one-default-content>>).
--
====
