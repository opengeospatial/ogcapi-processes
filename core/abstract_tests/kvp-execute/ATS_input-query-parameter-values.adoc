[[ats_kvp-execute_input-query-parameter-values]]

[abstract_test]
====
[%metadata]
identifier:: /conf/kvp-execute/input-query-parameter-values
target:: /req/kvp-execute/input-query-parameter-values
test-purpose:: Verify that the server can handle values passed by value and by reference in a KVP-encoded execute request.
test-method::
+
--
1. Get a list of processes as per test <<ats_core_process-list-op,/conf/core/process-list-op>>.

2. Retrieve the description of each process according to test <<ats_core_process-description,/conf/core/process-description>>.

3. Select a process to test and determine how values can be passed to each input of that process (i.e. by value, by reference or both, see the `valuePassing` parameter).

4. Composed a set of KVP-encoded execute requests according to <<<<req_kvp-execute_input-query-parameter-values,/req/kvp-execute/input-query-parameter-values>> to test each supported value passing method.

5. Execute each request using the HTTP `GET` method.

6. Verify that each response of the process is the expected response (see tests: <<req_core_process-execute-success-async,/req/core/process-execute-success-async>>, <<req_core_process-execute-success-async-outputs,/req/core/process-execute-success-async-outputs>>, <<req_core_process-execute-success-async-outputs-empty,/req/core/process-execute-success-async-outputs-empty>>, <<req_core_process-execute-success-async-outputs-omitted,/req/core/process-execute-success-async-outputs-omitted>>, <<req_core_process-execute-success-sync-empty-outputs,/req/core/process-execute-success-sync-empty-outputs>>, <<req_core_process-execute-sync-many-json,/req/core/process-execute-sync-many-json>>, <<req_core_process-execute-sync-one,/req/core/process-execute-sync-one>>, <<req_core_process-execute-sync-one-default-content,/req/core/process-execute-sync-one-default-content>>).
--
