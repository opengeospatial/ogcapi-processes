[[ats_kvp-execute_process-execute-op]]

[abstract_test]
====
[%metadata]
identifier:: /conf/kvp-execute/process-execute-op
target:: /req/kvp-execute/process-execute-op
test-purpose:: Verify that the server supports the HTTP GET operation at the `/processes/{processID}/execution` endpoint.
test-method::
+
--
1. Get a list of processes as per test <<ats_core_process-list-op,/conf/core/process-list-op>>.

2. Retrieve the description of each process according to test <<ats_core_process-description,/conf/core/process-description>>.

3. Select one or more processes to execute.

4. For each selected process, compose a KVP-encoded execute request as per <<req_kvp-execute_process-execute-op,/req/kvp-execute/process-execute-op>>.

5. Execute each KVP-encoded request using the HTTP `GET` method.

6. Verify that the response of each process is the expected response (see tests: <<req_core_process-execute-success-async,/req/core/process-execute-success-async>>, <<req_core_process-execute-success-async-outputs,/req/core/process-execute-success-async-outputs>>, <<req_core_process-execute-success-async-outputs-empty,/req/core/process-execute-success-async-outputs-empty>>, <<req_core_process-execute-success-async-outputs-omitted,/req/core/process-execute-success-async-outputs-omitted>>, <<req_core_process-execute-success-sync-empty-outputs,/req/core/process-execute-success-sync-empty-outputs>>, <<req_core_process-execute-sync-many-json,/req/core/process-execute-sync-many-json>>, <<req_core_process-execute-sync-one,/req/core/process-execute-sync-one>>, <<req_core_process-execute-sync-one-default-content,/req/core/process-execute-sync-one-default-content>>).
--
